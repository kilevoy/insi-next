# Parity для подкрановой балки

Модуль подкрановой балки пока развивается отдельно от общего калькулятора, поэтому доверие к нему держим через живое сравнение с Excel.

## Как проверять

Запуск:

```powershell
npm run crane-beam:compare
npm run crane-beam:compare:expanded
```

Скрипт:
- ищет workbook `подбор прокатной подкрановой балки v2.0.xlsx` в корне репозитория;
- при необходимости принимает путь через переменную `CRANE_BEAM_REFERENCE_WORKBOOK`;
- пересчитывает либо 10 контрольных сценариев, либо расширенную матрицу и сравнивает Excel с `calculateCraneBeam(...)`;
- пишет отчеты:
  - `docs/crane_beam_10_scenarios_comparison.json`
  - `docs/crane_beam_10_scenarios_comparison.md`
  - `docs/crane_beam_expanded_scenarios_comparison.json`
  - `docs/crane_beam_expanded_scenarios_comparison.md`

## Что считается опорой доверия

Пока модуль не встроен в общий калькулятор, считаем его надежным, если одновременно выполняются три условия:

- unit-тесты модуля зеленые;
- `npm run crane-beam:compare` дает полное совпадение по контрольным сценариям;
- `npm run crane-beam:compare:expanded` подтверждает расширенную сетку сценариев;
- изменения в логике проходят через `src/domain/crane-beam/**`, а не через обходные правки в UI.

## Что именно сравнивается

Для каждого сценария сравниваются:

- выбранный профиль;
- масса балки;
- коэффициент использования;
- усилия `Mx`, `My`, `Q`, `Qop`.

Этого достаточно, чтобы видеть и совпадение подбора, и совпадение ключевой расчетной механики с workbook.
